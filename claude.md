# Instrucciones del proyecto — OWD Site

## ⛔ INSTRUCCIÓN PRINCIPAL
**PROHIBIDO COMMITEAR O PUSHEAR SIN AUTORIZACIÓN EXPLÍCITA.**

- No ejecutar `git commit`, `git push`, ni ninguna operación que modifique el historial o el remoto sin que el usuario lo pida explícitamente en ese momento.
- La autorización para un commit/push NO se extiende a futuras acciones: cada commit y cada push requiere su propia confirmación.
- Sí está permitido: `git add`, `git status`, `git diff`, y preparar cambios — pero detenerse antes de commitear y esperar la orden.
